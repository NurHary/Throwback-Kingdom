# Logging Conventions
1.  Dalam penulisan logging, logging harus dipisah dimana harus dibuat chapter - chapter. setiap chapter berisikan beragam fiitur - fitur
    yang telah di implementasikan secara saling mendekat. biasanya satu chapter membahas pada levelan mereka sendiri - sendiri seperti contoh:
    chapter satu merupakan chapter pembangunan fundamental, sedangkan chapter kedua merupakan sistem pembangunan game loop
2.  Untuk setiap files dalam chapter harus memiliki hal sebagai berikut
    1.  Deskripsi Singkat terkait file ini membahas apa, tulis dalam format "_"
    2.  List Inspirasi yang menjadi inspirasi dalam pembangunan model tersebut
    3.  Main Line yang secara singkat memberikan alur dari file tersebut
    4.  Consideration untuk tempat brainstorming
    5.  Kesimpulan tempat untuk yang berisikan kesimpulan dari seluruh file tersebut
